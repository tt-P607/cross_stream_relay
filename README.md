# Cross Stream Relay (跨聊天流互通插件)

`cross_stream_relay` 是 Neo-MoFox 框架的插件，旨在打破不同聊天流（群聊、私聊）之间的信息孤岛。它赋予了 AI **“跨流感知”**与**“主动意识迁移”**的能力，使 AI 能够像人类一样，在群聊中被叫到后自然地转到私聊续上话题，或在不同群组间同步关键进展。

## 核心特性：跨流意识迁移 (Relay)

这是本插件最核心的能力，它模拟了人类在社交软件中切换窗口的行为逻辑。

### 1. 迁移动机与决策
AI 不再是被动等待消息，而是能根据当前对话上下文自主决策：
- **指令驱动**：用户明确要求“去私聊找我”或“去那个群帮我问问”。
- **价值驱动**：AI 发现话题涉及隐私、需要深度交流，或当前信息对另一个流有重大价值。

### 2. 完整的迁移链路
1. **目标寻址 (`find_target_stream`)**：AI 使用该工具通过名称、QQ号、群号或索引号反查目标流的元数据（`stream_id`, `target_id` 等）。
2. **意图封装 (`relay_to_stream`)**：
   - `relay_content`：AI 在目标流的开场白，包含来源说明和即时话题承接。
   - `opening_hint`：AI 给“未来的自己”留下的私密意识闪存，包含迁移目的和后续执行建议。
3. **冷流唤醒与注入**：
   - 插件自动检测目标流状态，若为冷态则调用 `StreamLoopManager` 强制启动。
   - 通过 `unread_messages` 机制注入虚拟系统消息，绕过常规适配器输入，实现“瞬时唤醒”。

### 3. 逻辑连贯性保障
- **身份保持**：在迁移过程中，插件会自动将请求者格式化为 `[昵称](ID)`，确保 AI 在新流中能立刻识别对方，避免“你是谁”的尴尬。
- **辅助提示词注入**：通过 `RelayMessageDecorator` 在目标流处理前注入辅助 Reminder，告知 AI 当前处于“跨流转告”状态，引导其自然承接逻辑。

## 辅助特性

### 1. 增量短期记忆 (Incremental Daily Memory)
- **Token 优化**：采用增量融合算法，每次更新仅处理“上次摘要 + 增量消息”，大幅降低长对话场景下的成本。
- **水位线机制**：精确记录已总结进度，确保记忆不遗漏、不重复。
- **第一人称视角**：强制 AI 以第一人称书写回忆，包含主观感受与未完成事项。

### 2. 跨流摘要 (Cross-Stream Summary)
- **全局感知**：通过 `SystemReminder` 将其他活跃流的摘要实时注入到当前对话中，使 AI 具备全局视野。
- **隐私控制**：支持 `off` / `one_way` / `two_way` 三种互通模式，严格保护私聊隐私。

## 开发者指南

### 核心组件

| 组件 | 类型 | 说明 |
| :--- | :--- | :--- |
| `find_target_stream` | **Tool** | **迁移前置**：反查目标流元组，获取 `stream_id`。 |
| `relay_to_stream` | **Action** | **迁移执行**：执行意识迁移，唤醒目标流并注入意图。 |
| `get_stream_raw_context` | **Tool** | **跨流查询**：查询其他流的原始记录（禁止查当前流）。 |
| `get_daily_memory` | **Tool** | **深度回溯**：跨群查询群聊的每日短期记忆。 |
| `AutoSummaryHandler` | **Event** | **自动维护**：监听消息流，维护各流摘要。 |

### 配置建议 (`config.toml`)
- 建议为 `daily_memory` 分配具备强推理能力的模型（如 `actor` 任务）。
- 在 `[relay]` 段中，可以通过 `usage_guide_prompt` 自定义 AI 的迁移动作指南。
